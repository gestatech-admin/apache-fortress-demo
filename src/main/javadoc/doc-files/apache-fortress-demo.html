Apache Fortress Demo

This section describes the installation and usage of the Apache Fortress Demo Web application

  1. Download the Apache Fortress Demo source bundle.

  2. Set JAVA_HOME: 
                    
    
# export JAVA_HOME=/opt/jdk1.7.0_10/
  3. Extract: 
                    
    
# $JAVA_HOME/bin/jar -xf apache-fortress-demo-master.zip
  4. Change directory: 
                    
    
# cd apache-fortress-demo-master/
  5. Set M2_HOME: 
                    
    
# export M2_HOME=/usr/local/maven-version
  6. Copy the fortress.properties, modified on step Apache Fortress Core SSL, to this package's resource folder. 
                        
    
# cp [directory-fortress-core]/src/test/resources/fortress.properties [apache-fortress-demo]/src/main/resources
    Where [directory-fortress-core] is base folder of the fortress core source package and [apache-fortress-demo] is the current package's home folder.

  7. Edit [apache-fortress-demo]/src/main/resources/fortress.properties file. Verify the host, port and ssl settings match Apache Fortress Core SSL setup.
    Update the JDBC configuration param section to match your setup from Install MySQL step. 
                        
    

# Use value from [Set Hostname Entry]:
host=fortressdemo2.com
port=10636

# These credentials are used for read/write access to all nodes under suffix:
admin.user=uid=admin,ou=system
admin.pw=secret

# This param tells fortress what type of ldap server in use:
ldap.server.type=apacheds

# Used for SSL Connection to LDAP Server:
enable.ldap.ssl=true
enable.ldap.ssl.debug=false

# Use truststore genned [Managing PKI Keys] or found under /src/test/resources in apache-fortress-demo source package.
trust.store=/home/smckinn/fortress/apache-fortress-demo/src/test/resources/mytruststore
trust.store.password=changeit

# Sets trust.store params as System.property to be used by JDBC driver:
trust.store.set.prop=true

# These JDBC config params must match settings from [Install MySQL] step:
database.driver=com.mysql.jdbc.Driver
database.url=jdbc:mysql://fortressdemo2.com:3306/demoDB?useSSL=true&requireSSL=true
database.username=admin
database.password=secret
  8. Save and exit.

  9. To build and deploy this web application, run this command: 
                        
    
# $M2_HOME/bin/mvn tomcat:deploy
  10. If you need to redeploy this web application, run this command: 
                        
    
# $M2_HOME/bin/mvn tomcat:redeploy
  11. Connect to web app:

    1. non-SSL - local: http://localhost:8080/apache-fortress-demo, remote: http://fortressdemo2.com:8080/apache-fortress-demo

    2. SSL - local: https://localhost:8443/apache-fortress-demo, remote: https://fortressdemo2.com:8443/apache-fortress-demo

      3. Accept the self-signed certificate when the warning dialog appears.

  12. Use Apache Solenium to drive Firefox through a range of security unit tests. Firefox must be installed before running this step. 
                    
    
# $M2_HOME/bin/mvn verify -DskipTests=false
