OpenLDAP SSL



This section enables OpenLDAP SSL.

Steps to Enable

  1. use artifacts created during Managing PKI Keys for OpenLDAP ssl.

  2. Update openldap's config to point to these files and they must be accessible by slapd process at runtime: 
                        
    
### SSL Configuration
TLSCACertificateFile  /path/to/ca-cert.pem
TLSCertificateFile    /path/to/server-cert.pem
TLSCertificateKeyFile /path/to/server-key.pem
  3. Modify the slapd startup script to include LDAPS port (default 636) upon startup. e.g. 
                        
    
"ldap:/// ldaps:///"
  4. Restart the slapd process.

  5. Check the status of server process. From system command prompt (as apacheds user): 
                
    
[student@NY1SCOLFTDEMO01 directory-fortress-core]$ ps -ef | grep slapd
root      2144     1  0 13:17 ?        00:00:00 /opt/symas/lib64/slapd -u root -g root -h ldap:/// ldaps:/// ldapi:///
student   2185  2022  0 13:34 pts/2    00:00:00 grep --color=auto slapd
    Verify slapd server restarted successfully and is now listening over LDAPS port.
